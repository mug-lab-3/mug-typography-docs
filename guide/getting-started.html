使い方 ​

本プラグインは、入力したいテキストの長さや文字の複雑さに合わせて、処理の制限値（最大文字数・最大パーツ数）が異なる3つのプリセットに分かれています。用途に合わせて選択してご使用ください。

  • Mug Typography (Low): 入力するテキストが短い場合（最大16文字 / 60パーツまで）に使用します。動作が最も軽快です。
  • Mug Typography (Mid) [推奨]: 通常のテキスト入力向けです（最大30文字 / 84パーツまで）。どれを使用すればよいか迷った場合は、まずこの「Mid」を選択してください。
  • Mug Typography (High): 長文テキストや、画数の多い複雑な漢字を多く含める場合（最大60文字 / 170パーツまで）に使用します。制限値が大きいぶんメモリ使用量も多く、搭載メモリの少ない環境ではクラッシュのリスクが高くなります。
  • Mug Typography Lite (Low / Mid / High): 個別パーツごとの調整パラメータ（Part Settings）を排除してロードと処理を高速化している軽量版プリセットです。個別パーツの編集が不要な場合はこちらを推奨します。
    • 制限値の自動緩和: 最大文字数が通常版の 1.5倍 に自動的に緩和されます（LiteLow: 最大24文字, LiteMid: 最大45文字, LiteHigh: 最大90文字）。
    • 最大パーツ数の上限解放: 個別パラメータを保持しないため、描画できる上限である 512パーツ まで制限なく許容されます。

IMPORTANT

通常は Low もしくは Mid (またはそれらの Lite 版) を使用し、文字数・パーツ数が足りない場合にのみ High / LiteHigh を使用してください。 High は制限値が大きいぶんメモリを多く消費するため、搭載メモリの少ない環境では DaVinci Resolve ごとクラッシュする恐れがあります。 もし High / LiteHigh でクラッシュする場合は、preferences.json で High の maxCharacters / maxParts を実際に必要な値まで下げることで、安定して使用できる場合があります（調整方法は付録の「環境設定のカスタマイズ (preferences.json)」を参照）。

Edit ページ ​

エフェクトライブラリの Open FX には、用途に合わせて以下の2つの場所に上記の各プリセット（Low / Mid / High / Lite Low / Lite Mid / Lite High）が用意されています。タイムラインに配置または適用してご使用ください。

  • Open FX → Generators → MugLab (ジェネレーター版)
    • タイムライン上に新規クリップとして直接配置したい場合に使用します。ドラッグ＆ドロップでタイムライン上に配置できます。
  • Open FX → Filters → MugLab (フィルター/エフェクト版)
    • 既存のクリップに対して、エフェクトとして本プラグインを適用したい場合に使用します。
    • なお、Color ページ などの他のページではこちら（Filters）のみが表示されますので、それを選択してご使用ください。
    • ※ グラフィックボードのエラー等によりCPU描画に切り替わった場合は、仕様上背景との合成がスキップされ、テキストのみが表示される状態になります（詳細は付録の「制限事項」を参照してください）。

配置または適用後、右側の Inspector パネルにパラメーターが表示されます。

ビューア上での操作を有効にする ​

プレビュー画面上でのハンドル操作（移動・回転・スケールなど）を使うには、タイムライン左下の四角く囲った部分を クリック → Open FX Overlay に設定してください。

NOTE

Edit ページには DaVinci Resolve の仕様による制限があります。詳細は付録の「制限事項」を参照してください。

Fusion ページ ​

Fusion ページでは、メニューの Tools → OpenFX → MugLab 配下の Mug Typography (Low), Mug Typography (Mid), Mug Typography (High) から必要なプリセットのノードをノードグラフに追加して使用します。ノードを選択すると右側の Inspector にパラメーターが表示されます。

NOTE

Fusion ページでは、入力（Input）端子に映像を接続すればフィルターとして動作し、接続しない場合はジェネレーターとして機能します。ただし、パフォーマンスの観点から接続しての使用は推奨しません（また、CPU描画に切り替わった場合は、背景との合成がスキップされ、テキストのみが表示されます）。
