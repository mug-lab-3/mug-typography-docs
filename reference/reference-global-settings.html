Page: Global Settings ​

テキスト設定（テキスト・フォント・文字組み・モーフアニメーション）・全体変形・カラー設定（塗り色・グラデーション）・輪郭線（Stroke）・ドロップシャドウ・バウンディングボックスを設定するページです。

リセット ​

パラメーター説明
Reset All Charactersすべての文字とパーツの Pos / Pivot / Scale / Stretch / Rotation / Margin After / Use Indiv Color / Color / Use Indiv Stroke / Stroke Width / Stroke Gap / Stroke Color をデフォルト値に戻します。

NOTE

リセットの対象は 文字・パーツ単位の個別設定のみ です。Global Settings（テキスト全体に適用される設定）はリセットされません。

テキスト設定（Text Settings） ​

テキストの内容・フォント・文字組み（配置・間隔・全体スケール）をまとめて設定するグループです。

Fontリストから使用するフォントを選択します。-書体（デザイン）を変更したい場合。

・Center: テキストのインクバウンディングボックス（描画される範囲）の中心を基準にします。
・Bottom: テキストの下端を基準にします。
・Baseline: テキストのベースライン（基準線）を基準にします（文字を追加・変更した際の上下のズレや、モーフアニメーション時の垂直ブレを防ぐのに適しています）。		Centerテキストの垂直方向の基準点を設定したい場合（複数行テキストや縦書きテキストで効果が顕著です）。

NOTE

複数行テキストの入力と改行について

  • Fusionページでの入力: Fusionページでは、Enterキーを押すことで直接改行を入力して複数行のテキストを描画できます。

  • Editページでの入力: Editページのインスペクタの制限により、テキストボックス内で直接Enterキーによる改行が入力できません。このため、Editページで複数行を入力したい場合は、改行したい位置に \n または <br> と入力してください。これらは自動的に改行として扱われます。

    また、Editページではテキスト入力中に Enter キーを押すと、改行ではなくフォーカスが別のパラメーターへ移動してしまうことがあります。入力中に誤って Enter キーを押さないようご注意ください。

  • 改行エスケープ: 改行文字としての動作を無効にして \n や <br> をそのままテキストとして表示したい場合は、直前にバックスラッシュ（Windows環境では円マーク）を付加して \\n または \<br> と入力してください。

テキスト入力の記述描画結果
あ\nいあ
い (2行で描画されます)
あ<br>いあ
い (2行で描画されます)
あ\\nいあ\nい (エスケープされ、改行されずに1行で表示されます)
あ\<br>いあ<br>い (エスケープされ、改行されずに1行で表示されます)

✨ モーフアニメーション（Morph Animation） ​

Text Settings の Text 欄のすぐ下にある、モーフアニメーション用のグループです。Text に入力した文字列のアウトラインから、別の文字列のアウトラインへなめらかに変形（モーフ）させるアニメーションを作成できます。「A」が「B」に溶けるように変わる、といった表現が可能です。

基本的な使い方:

  1. Enable Morph を ON にします。
  2. Text（変形の 開始 となる文字列）を入力します。
  3. To Text に、変形の 終了 となる文字列を入力します。
  4. Progress にキーフレームを打ち、0（開始テキスト）から 1（終了テキスト）へ変化させます。
パラメーター説明デフォルト
Enable Morphモーフアニメーションを有効にします。ON にすると、通常のテキスト描画の代わりに、Text（開始）から To Text（終了）へのモーフ描画が行われます。OFF
To Text変形の終了地点となるテキストを入力します。複数行に対応しています。(空)
Progress変形の進み具合を 0.0（＝ Text の状態）から 1.0（＝ To Text の状態）で指定します。この値をタイムライン上でキーフレームすることで、変形の速度やタイミングを自由にコントロールできます。イージング（緩急）もこのキーフレームのカーブで調整します。0.0
Matching Strengthパーツのマッチング（どのパーツとどのパーツを対応させるか）を、単純な位置の近さだけでなくより賢く調整する強さを指定します。0.0 では位置のみで対応付け（従来の挙動）、値を上げると次の2つが働きます。①塗り面積の類似度を考慮し、面積が近いパーツ同士を優先的にペアにします（点などの小さなパーツが太いストロークと対応してしまうのを防ぎます）。②個別の色・枠・影を設定した文字を、To Text 側の同じ位置の文字へ優先的に対応させ、その個別スタイルが無関係な文字に取り残されるのを防ぎます。0.3
Same Glyph Priority開始テキストと終了テキストで同じ字形の文字が含まれる場合に、その文字同士を優先的にペアにする強さを指定します。0.0 では位置のみで対応付け、1.0 では距離に関わらず常に同じ字形同士を対応させます。同じ文字が含まれる場合、その部分はモーフせずスライドするだけになるため、変形をすっきりさせたいときに有効です。0.3

NOTE

Morph が ON の間は、通常のテキスト描画に代わってモーフ専用の描画に切り替わります。変形元のテキストは、上の Text 欄が使われます。

WARNING

モーフアニメーションでは、変形の前後で文字数やパーツ数に違いがある場合、文字・パーツ個別の色・枠・影の設定が、モーフ OFF 時の見え方と一致しないことがあります。そのため、モーフアニメーションでは文字・パーツごとの個別の色・枠・影設定の使用は非推奨です。

TIP

モーフの出力結果にも、パス変形エフェクト（パルス・台形・球面化・樽型）や手描き風エフェクトをそのまま重ねて適用できます。

TIP

開始テキストと終了テキストで文字数が異なっていてもモーフできます（足りない分の文字は自動で出現・消滅します）。ただし、文字数や文字の形の複雑さ（パーツ数）が近いテキストどうしのほうが、より自然できれいに変形します。

全体変形（Transform） ​

テキスト全体に一括で適用する位置・回転・ストレッチを設定するグループです。

パラメーター説明デフォルトユースケース
Positionキャンバス上におけるテキスト全体の表示位置を指定します。0.5, 0.5 がキャンバスの中心です。0.5, 0.5テキストをフレーム内の特定の位置に配置する場合。
Pivotキャンバス上における全体変形の拡大縮小・回転の基準点（ピボット）を指定します。0.5, 0.5回転や拡大縮小の軸を端部など任意の位置にずらしたい場合。
Rotationテキスト全体の回転角度を指定します。0°テキスト全体を傾けたい場合。
StretchX 方向と Y 方向の比率を個別に拡大縮小します。1.0, 1.0ロゴデザインのように、テキストを意図的に縦長または横長に歪ませたい場合。

NOTE

全体の拡大縮小（Scale）は、テキスト設定（Text Settings）グループ内にあります。Scale は縦横同時にサイズを変更するのに対し、Stretch は片方の軸のみを変化させるため、テキストを縦方向や横方向に変形させるのに適しています。

カラー設定（Color Settings） ​

テキストの塗り色・透明フィル・グラデーションをまとめて設定するグループです。

パラメーター説明デフォルトユースケース
Font Colorテキスト全体の基本色（RGB + 不透明度）を指定します。-テキスト全体の色を一括で設定したい場合。グラデーション適用時は開始色として機能します。
Transparent FillON にすると、テキスト本体（塗り＝フィル）を完全に透明にし、輪郭線（Stroke）だけを描画します。OFF中抜き文字（アウトラインのみの文字）を作りたい場合。輪郭線（Stroke）の Width を 0.0 より大きく設定して併用します。なお、Character / Part 側で個別カラー（Use Indiv Color）を指定した文字・パーツには適用されません。

（Transparent Fill を ON にし、Stroke の Width / Gap を設定して作成した中抜き文字の例）

グラデーション（Gradient） ​

カラー設定（Color Settings）グループ内のサブグループです。Font Color（塗りの色）を開始色として、テキスト全体にグラデーションを適用します。Color Space が None 以外の値に設定されている場合にグラデーションが有効になります。

NOTE

グラデーションの適用範囲は、**すべてのパーツの変形後の位置を囲むバウンディングボックス（BB）**を基準に計算されます。文字やパーツを移動・拡大縮小して BB の大きさが変化すると、それに伴ってグラデーションの適用範囲も自動的に変化します。

パラメーター説明デフォルト
Color Spaceグラデーションのカラー補間モードを選択します。None で無効化されます。Okhsv / Okhsl は色相を維持した知覚的に均一な補間を行います。Oklab は知覚的な明度空間で補間します。Linear RGB は線形 RGB 空間で補間します。None
Gradient End Colorグラデーションの終了色（RGB + 不透明度）を指定します。Font Color が開始色になります。オレンジ (#ff8000・不透明)
Midpoint開始色と終了色の 50/50 のブレンド位置を指定します（0.5 = 均等に分割）。0.5
Angleグラデーションの方向を角度で指定します。0°が左から右、90°が上から下、270°が下から上です。270°
Startバウンディングボックスに対するグラデーションの開始位置を指定します（0 = BBの開始位置、1 = BBの終了位置）。0.2
Endバウンディングボックスに対するグラデーションの終了位置を指定します（0 = BBの開始位置、1 = BBの終了位置）。0.8

Color Space ごとの補間の違い ​

同じ2色（開始色 #0000FF＝青、終了色 #FFFF00＝黄）でも、選択する Color Space によって中間色の見え方が大きく異なります。Okhsv / Okhsl は色相環に沿って鮮やかな中間色（シアン〜緑）を通り、Oklab は知覚的に均一な明度で、Linear RGB は単純な数値補間のため中間が濁ったグレーになります。

輪郭線（Stroke） ​

テキストグリフの輪郭線（ストローク）を描画します。複雑な背景の上でテキストの視認性を保ちたい場合や、文字に太い縁取りを付けたい場合に便利です。

TIP

縁取りを表示するには、まず Width（太さ）を 0.0 より大きい値に設定してください。0.0 のままでは縁取りは描画されません。

パラメーター説明デフォルト
Width輪郭線の太さを、フォントサイズに対する比率で指定します（例: 0.05 で文字の高さの約 5%）。スケール・ストレッチ変形には追従しません。0.0 に設定すると輪郭線描画は無効化されます。0.0
Gap塗り（フィル）と輪郭線（ストローク）の間に空ける隙間の幅を、フォントサイズに対する比率で指定します。0.0 より大きい値にすると、文字本体と輪郭線の間が一度くり抜かれ、本体・隙間・輪郭線の3層に分かれた「中抜き縁取り」のような表現になります。0.0
Color輪郭線の色（RGB + 不透明度）を指定します。オフブラック (#1c1c1e)
Order輪郭線と塗りの描画順序を指定します。Fill Over Stroke（ストロークの上に塗り）では塗りがストロークを上書きするため、外側半分のみが見え輪郭線が外側に広がる効果になります。Stroke Over Fill（塗りの上にストローク）ではストロークがグリフに重なり内側に食い込んで見えます。Fill Over Stroke
Join Style輪郭線の角の接続（ジョイン）スタイルを指定します。
・Miter Clip: マイター接続。角を尖らせますが、マイター制限を超えた場合はクリップ（平坦化）されます。
・Round: 角を丸めます。
・Bevel: 角を平らにします（面取り）。
・Miter Round: マイター接続ですが、制限を超えた場合は丸めます。
・Miter Bevel: マイター接続ですが、制限を超えた場合は平らにします。		Miter Clip
Miter Limitマイター接続における角の尖り方の制限値です。角が鋭角になるほどマイター長（尖り具合）が長くなりますが、この制限値を超えた場合は、Join Style で指定されたフォールバック処理（クリップ、丸め、面取りなど）が適用されます。4.0

シャドウ設定（Shadow Settings） ​

テキスト全体にドロップシャドウを描画します。文字を背景から浮き立たせて立体感を出したいときや、視認性を高めたいときに使います。

パラメーター説明デフォルト
Draw Global Shadowすべての文字にドロップシャドウを描画する、グローバルのシャドウを有効にします。ON にすると以下のパラメーターが表示されます。
※ Character Settings 側で文字ごとに個別シャドウカラー（Use Indiv Shadow Color）を指定した文字は、このグローバルシャドウが OFF でも影を落とします。		OFF
Distanceシャドウをずらす距離を、キャンバスの短い方の辺（縦横のうち短い辺）に対する % で指定します。1.0
Angleシャドウの方向を角度で指定します。0° が右、90° が下方向です。45°
Colorシャドウの色（RGB + 不透明度）を指定します。青 (#5555ff・不透明)

バウンディングボックス設定（Bounding Box Settings） ​

テキスト全体を囲むバウンディングボックス（背景枠）を描画・塗りつぶしする設定です。字幕の背景に帯状の下地を敷きたいときや、テキストを枠で囲んでテロップ風にしたいときに使います。

NOTE

バウンディングボックス（BB） とは、すべての文字・パーツを囲む最小の四角い範囲のことです。文字やパーツを個別に動かすと、それに合わせて BB の大きさ・位置も変化します。

パラメーター説明デフォルト
Draw Bounding Boxバウンディングボックス背景・枠線の描画を有効にします。これを ON にすると、以下のパラメーターが表示されます。OFF
Top Edge / Bottom Edge / Left Edge / Right Edgeそれぞれ上・下・左・右のどの辺を描画するかをON/OFFで指定します。ON（すべての辺）
Border Width境界線の太さを、フォントサイズに対する % で指定します。20.0
Border Color境界線の色を指定します。赤 (1, 0, 0, 1)
Fill Backgroundボックス内部を背景色で塗りつぶすかどうかを指定します。ON にするとグリフ（文字）の背後に背景が描画されます。OFF
Fill Color背景塗りつぶし時の色（RGB）を指定します。薄紫 (1, 0.667, 1, 1)
Fill Opacity背景塗りつぶしの不透明度を指定します（0.0 = 完全透明、1.0 = 完全不透明）。1.0
Start Cap Style / End Cap Style描画する辺が3つ以下の開いたパス（例: L字型など）の時、描画する枠線の開始点・終了点の端部スタイル（Cap Style）を指定します。選択肢: Butt (平ら), Square (四角), Round (丸), Round Rev (逆丸), Triangle (三角), Triangle Rev (逆三角)。Butt
Margin X / Margin Yボックス左右(X)・上下(Y)の余白（マージン）を、フォントサイズに対する % で指定します。負の値を指定して文字の内側に枠線を引くことも可能です。40.0
Corner Radiusバウンディングボックス背景の角を丸める半径をフォントサイズに対するパーセンテージで指定します。Top / Bottom / Left / Right の全 Edge が ON になっている場合のみ適用されます（いずれかの辺が OFF の場合は無視されます）。0.0
