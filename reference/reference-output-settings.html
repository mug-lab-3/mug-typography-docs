Page: Output Settings ​

特定のパーツや文字だけを別の出力として分離・抽出したり、一部の要素だけを描画したりするための設定ページです。
この機能を利用することで、同一テキストの複数インスタンスに対してそれぞれ個別のパーツだけを透明背景で出力させ、後段のノードで異なるカラー・エフェクトを個別に合成するような高度なコンポジットが可能になります。

パラメーター ​

パラメーター説明デフォルト
Enable Output Separation出力分離機能を有効にします。ON にすると、以下のパラメーターが表示されます。OFF
Output Mode分離対象をどのように描画するかを指定します。以下の3つから選択可能です。
・Default: Separation 1・Separation 2 のどちらにも含まれていないパーツをレンダリングします（バウンディングボックス背景も描画されます）。
・Separation 1 Only: Separation 1 Targets で指定したパーツ「のみ」をレンダリングします（バウンディングボックス背景は描画されません）。
・Separation 2 Only: Separation 2 Targets で指定したパーツ「のみ」をレンダリングします（バウンディングボックス背景は描画されません）。		Default
Separation 1 Targets第1分離グループの対象とする文字・パーツを指定するテキスト入力欄です。記述方法は下記の「Targets（対象指定）の記述フォーマット」を参照してください。
空欄の場合、その分離グループには何も含まれません（＝そのグループは対象なしになります）。		(空)
Separation 2 Targets第2分離グループの対象とする文字・パーツを指定するテキスト入力欄です。記述方法は Separation 1 Targets と同じです（空欄の場合は何も含まれません）。(空)

IMPORTANT

分離ターゲット（Separation 1 / 2 Targets）では、空欄＝そのグループには何も含まれない（対象なし） という扱いになります。後述するパス変形・手描き風エフェクトの Targets（空欄＝すべてが対象）とは空欄時の意味が逆になる点にご注意ください。

TIP

ワークフロー例

  1. エフェクトノードをコピーして3つ作成します。
  2. 1つ目のノードで Enable Output Separation を ON、Output Mode を Default にし、Separation 1 Targets に c1、Separation 2 Targets に c2 を指定します（1・2文字目以外が描画されます）。
  3. 2つ目のノードで Enable Output Separation を ON、Output Mode を Separation 1 Only にし、Separation 1 Targets に c1 を指定します（1文字目だけが描画されます）。
  4. 3つ目のノードで Enable Output Separation を ON、Output Mode を Separation 2 Only にし、Separation 2 Targets に c2 を指定します（2文字目だけが描画されます）。
  5. 各ノードに個別のエフェクトを適用してマージすることで、文字ごとに異なるエフェクトを持つアニメーション演出が実現できます。

✨ Write On（出現フィルター） ​

Write On は、テキストの文字またはパーツを、先頭から末尾へと段階的に表示・非表示にするフィルター機能です。
Start と End パラメーターをタイムラインでキーフレームすることで、文字が順番に現れていく「書き文字」風アニメーションを簡単に作成できます。

パラメーター説明デフォルト
Unitフィルターの単位を選択します。
・Character: 文字（キャラクター）単位でフィルタリングします。Start〜End の範囲に含まれる文字のみ描画されます。
・Part: パーツ単位でフィルタリングします。Stroke Order との組み合わせで、書き順に近い順序でパーツを出現させられます。		Character
Start表示範囲の開始位置を 0.0（先頭）〜 1.0（末尾）の割合で指定します。アニメーション可能。0.0
End表示範囲の終了位置を 0.0（先頭）〜 1.0（末尾）の割合で指定します。アニメーション可能。1.0
Stroke OrderON にすると、各文字のパーツを出現順序が自然に見えるよう並び替えます。Unit が Part のときに特に有効で、おおよそ書き順に近い順序でパーツが現れるライトオン効果を実現します。OFF

TIP

Write On アニメーションの作り方

  1. Unit を Part に設定し、Stroke Order を ON にします。
  2. アニメーションの開始フレームで End を 0.0 にキーフレームします。
  3. アニメーションの終了フレームで End を 1.0 にキーフレームします。
  4. これだけで、文字を書いていくようなアニメーションが完成します。

NOTE

Start と End が両方デフォルト値（Start = 0.0、End = 1.0）の場合、Write On フィルターは無効（すべて表示）として処理されます。

IMPORTANT

Stroke Order の出現順序について
Stroke Order による並び替えは、本来の正しい筆順（書き順）を再現するものではありません。パーツの位置関係などからそれらしい順序を内部アルゴリズムで推定して並べているもので、実際の書き順とは異なる場合があります。また、パーツへの分かれ方は使用するフォントのデータ構造によって決まるため（概要を参照）、同じ文字でもフォントが変われば出現順が変わることがあります。意図した順序になるかは、実際のフォントでプレビュー確認してください。

Targets（対象指定）の記述フォーマット ​

Output Separation の Separation 1 / 2 Targets、およびパス変形エフェクト・手描き風エフェクトの Targets は、いずれも同じ記述フォーマットで対象の文字・パーツを指定します。

トークン（指定の単位）を カンマ(,) またはスペースで区切って複数並べます。各トークンは以下の表記が使用できます（大文字・小文字は区別されません。c1 と C1 は同じ）。

表記意味例
c<n>文字単位。n 文字目（1始まり）を構成するすべてのパーツを指定します。c1 = 1文字目のすべてのパーツ
p<n>パーツ単位。グローバルで n 番目（1始まり）のパーツを指定します。p5 = 5番目のパーツ
<n>プレフィックスなし。p<n> と同じく、n 番目のパーツを指定します。15 = 15番目のパーツ（p15 と同じ）
<a>-<b>範囲指定。a から b まで（両端を含む）をまとめて指定します。詳細は下記。c1-c3, p2-p5, 2-5
!<トークン>否定（除外）指定。先頭に ! を付けると、そのトークンが選ぶ対象を除外します。詳細は下記。!c2, !p3, !2-5

範囲指定（-）の詳しい挙動 ​

  • 範囲の両端は同じ種類（文字どうし、またはパーツどうし）である必要があります。種類が混在した指定（例: c1-p3）は無効として無視されます。
  • 片方の端だけにプレフィックスを付ければ十分です。 プレフィックスのない端は、もう一方の端の種類を引き継ぎます。
    • c1-3 は c1-c3 と同じ（1〜3文字目）として扱われます。c1-c3 とすべて書く必要はありません。
    • 1-3 は p1-p3 と同じ（1〜3番目のパーツ）として扱われます。
    • 引き継ぎは逆順でも働きます。c3-1 は c3-c1 → 並べ替えて c1-c3 になります。
  • 開始と終了が逆順（例: c3-c1）でも、自動的に昇順に並べ替えて解釈されます。

否定指定（!）の詳しい挙動 ​

  • ! を付けたトークンは、その対象を結果から取り除きます。範囲や文字単位にも使えます（例: !c2, !p3-p5）。
  • トークンの順序は問いません。 すべての通常指定と除外指定を集計したうえで、除外が優先されます（同じパーツが指定と除外の両方に該当した場合は対象から外れます）。
  • 否定指定は、各エフェクトの「空欄時の挙動」と組み合わせて使うと便利です（下記参照）。

空欄時の挙動（重要・機能ごとに逆） ​

何も入力しなかった場合の扱いは、機能によって逆になります。

機能空欄時の意味
パス変形エフェクト / 手描き風エフェクト（Targets）すべての文字・パーツが対象になります。
Output Separation（Separation 1 / 2 Targets）何も含まれません（対象なし）。

このため、パス変形・手描き風エフェクトで「特定のパーツ以外すべて」を対象にしたい場合は、否定指定だけを書くのが便利です。たとえば !c1 と入力すると、1文字目を除くすべてが対象になります（空欄＝全部 から 1文字目だけ除外されるため）。

記述例 ​

入力解釈
c1, p12, 151文字目のすべてのパーツ、12番目のパーツ、15番目のパーツ
c1-c31〜3文字目のすべてのパーツ
c1-3c1-c3 と同じ（1〜3文字目）。終端のプレフィックスは省略可
p2-p5 / 2-52〜5番目のパーツ（どちらも同じ）
c1-5 !c31〜5文字目のうち、3文字目を除いたすべて
!p3（変形エフェクト側）3番目のパーツを除く、すべてのパーツ

NOTE

0以下の値や、存在しない範囲外のインデックス、種類が混在した範囲などの無効な指定は黙って無視されます（エラーにはなりません）。
