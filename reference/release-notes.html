リリースノート (Release Notes) ​

v3.2j ​

  • [不具合修正] リセット時にまれにクラッシュする問題 — パラメーターをリセットした際に、まれに DaVinci Resolve がクラッシュすることがある不具合を修正しました。

v3.2h / v3.2i ​

  • [✨ 新機能] Write On — 文字・パーツを順番に出現させる — Output Settings に Write On 機能を追加しました。Start / End の値をキーフレームするだけで、テキストの文字やパーツを先頭から末尾へ順番に出現・消滅させられます。Stroke Order と組み合わせれば、文字を1画ずつ書いていくような「ライトオン」アニメーションも手軽に作成できます。（詳しくは Write On（出現フィルター） を参照）

v3.2g ​

  • [不具合修正] モーフ中のパス変形・手描き風エフェクトの対象指定（Targets）の追従 — モーフアニメーションの再生中、パス変形エフェクト（パルス・台形・球面化・樽型）および手描き風エフェクトの対象指定（Targets。例: c2, p3）が、意図した文字・パーツではなく無関係なパーツに適用されてしまう不具合を修正しました。対象指定を変形元（Text）・変形先（To Text）の両方で解決するようにしたことで、c2 / p3 などの指定がモーフ中も対象のグリフに正しく追従するようになりました。

v3.2f ​

  • [品質改善] モーフのマッチング精度向上 — モーフアニメーションのパーツ対応付けで、装飾（色などのスタイル）を設定した文字を、変形先の同じ位置の文字へ優先的に対応させるよう改善しました。スタイルを付けた文字がモーフ中に意図しないパーツへ移動するのを抑え、より自然な変形になります。

v3.2 (v3.2a / v3.2b / v3.2c / v3.2d / v3.2e) ​

  • [新機能] モーフアニメーション（Morph Animation） — テキストAからテキストBへ、文字が溶け込むように滑らかに変形する「モーフアニメーション」機能を追加しました。Progressパラメータをキーフレームすることで、変形速度やタイミングを自在にコントロールできます。
  • [新機能] 垂直アライメントに「Baseline」を追加 — 垂直方向の揃え位置（Vertical Alignment）に「Baseline（ベースライン）」オプションを追加しました。文字の追加や変更の際にテキストが上下にガタつくのを防ぎ、モーフアニメーション時も垂直方向にブレずに安定した変形が行えるようになります。
  • [新機能] パラメーターのコピー／ペースト（Copy / Paste） — 文字やパーツの変形パラメーター（位置、回転、スケールなど）を、別の文字やパーツへコピー＆ペーストできる機能を追加しました。複数文字の調整を大幅に効率化できます。
  • [新機能] アウトライン表示のショートカット — プレビュー画面で文字の境界線（アウトライン）の表示／非表示を、ショートカットキー Alt+;（macOS: Option+;）で素早く切り替えられるようになりました。
  • [変更] グローバルシャドウと個別シャドウカラーの関係 — 特定の文字だけに個別の影色を設定（Use Indiv Shadow Color）した際、全体の影（Draw Global Shadow）が OFF の状態でも、その文字だけが影を落とすように仕様を改善しました。特定の文字だけ影を付けたいという表現がより簡単に作れます。
  • [変更] 球面化・樽型歪みエフェクトの値域拡張 — 「球面化（Sphere）」「樽型歪み（Barrel）」エフェクトの変形量（Amplitude）について、数値入力による設定範囲を -10.0〜10.0 まで大幅に拡張しました。よりダイナミックな歪み効果を表現できます。
  • [品質改善] レンダリングパフォーマンスの最適化 — 描画キャッシュシステムの導入や内部的なメモリ管理の最適化により、プレビュー再生時のカクつきを抑え、全体的な処理パフォーマンスとメモリ効率を向上させました。

v3.1a ​

  • [新機能] 中抜き文字（Transparent Fill） — Global Settings に Transparent Fill パラメータを追加しました。ON にすると文字本体（塗り）を完全に透明にし、輪郭線（Stroke）だけを描画できます。輪郭線の Width を設定して併用することで、アウトラインのみの中抜き文字を作成できます。
  • [新機能] ストロークギャップ（Stroke Gap） — 塗り（フィル）と輪郭線（ストローク）の間に隙間を空ける Gap パラメータを、グローバルの輪郭線設定と文字ごとの個別輪郭線設定の両方に追加しました。本体・隙間・輪郭線の3層に分かれた「離れた縁取り」のような表現が可能です。
  • [新機能] キーボードによる文字の透明化／復元 — Character 選択モードで、Alt+Ctrl+[（macOS: Option+Command+[）で選択中の文字を透明化（アルファをクリア）、Alt+Ctrl+]（macOS: Option+Command+]）で透明化を解除してグローバルカラーに戻せるようになりました。これらのショートカットのキーコードは preferences.json の clearAlpha / restoreAlpha でカスタマイズ可能です。
  • [変更] プリセットの最大パーツ数の調整 — メモリ負荷のバランスを見直し、各プリセットの最大パーツ数を調整しました（Low: 64→60, Mid: 90→84, High: 180→170）。最大文字数は従来どおりです（Low: 16, Mid: 30, High: 60）。引き続き preferences.json で各上限値をカスタマイズできます。

NOTE

プリセットの制限値はプラグインを新規に配置した際の初期値です。preferences.json をすでにカスタマイズしている場合は、その設定値が優先されます。

v3.0c ​

  • [品質改善] 球面化・樽型歪みエフェクトの変形がより滑らかに
    • 文字のサイズが非常に大きい場合や、大きくズームした際にも、歪んだ輪郭がカクカクせず常に滑らかなカーブを描くように描画処理を改善しました。
  • [品質改善] 描画の安定性とパフォーマンス向上
    • GPU（CUDA）を使用した描画時のメモリ管理を見直すことで、動作の安定性と処理速度がさらに向上しました。

v3.0a ​

  • [新機能] 4種類の新規パス変形（Path Deformation）エフェクトの追加 — 文字のアウトラインを直接伸縮・歪曲させるアニメーションエフェクトが追加されました。
    • Pulse (パルス/呼吸): 文字の輪郭を外側・内側へ膨張・収縮させるアニメーション。
    • Trapezoid (台形変形): 指定した角度の軸に沿って台形にすぼませる歪み効果。
    • Sphere (球面化/バルーン): 文字を球面状に膨らませる／すぼませる効果。
    • Barrel (樽型歪み): 文字の中心から外側へスケールを徐々に変化させる、樽型・糸巻き型の歪み効果。
    • いずれのエフェクトも共通して、速度・位相・文字ごとの位相（ウェーブ効果）・開始遅延、および適用する対象パーツ（Targets）の個別指定に対応しています。
  • [新機能] 3種類の Lite プリセット（LiteLow, LiteMid, LiteHigh）の追加 — 個別パーツパラメータ（Part Settings ページおよび関連パラメータ）を排除することでプラグインのロードと処理を大幅に高速化した軽量版プリセットが追加されました。
    • メリット: 個別パーツパラメータがない代わりに、最大文字数が通常版の 1.5倍 (LiteLow: 24文字, LiteMid: 45文字, LiteHigh: 90文字) に自動的に緩和され、かつ最大パーツ数も上限である 512パーツ まで上限を意識せず使用できます。
  • [新機能] 文字の整列（Align Selected）機能の追加 — ビューア上で複数選択した文字を一括で整列（パッキング）させることができるボタンが Character Settings ページに追加されました。あわせて、DaVinci Resolve のデフォルトショートカットと衝突しないショートカットキー Alt+Ctrl+H（水平整列） / Alt+Ctrl+V（垂直整列）が追加されました。
  • [新機能] 選択モード切り替えショートカットの追加 — ビューア上での操作対象レベル（Global / Character / Part）をキーボードから素早く切り替えるためのショートカットキー Alt+Ctrl+A（Global）/ Alt+Ctrl+C（Character）/ Alt+Ctrl+P（Part）が追加されました。（※Lite版では Part 選択モードは無効になります）
  • [改善] キーボードショートカットのカスタマイズ機能 — preferences.json に shortcuts オブジェクトを追記することで、整列、選択モード切り替え、キーフレーム打刻などのショートカットの基本キーコードをユーザー好みにカスタマイズできるようになりました。
  • [改善] 手描き風エフェクトに対象指定（Targets）パラメータを追加 — 手描き風エフェクト（Hand Drawn Effect）をすべての文字ではなく、指定したインデックスの文字やパーツ（例: 濁点のみ、特定の文字のみ）にだけ適用できるようになりました。
  • [改善] パフォーマンスの大幅な向上 — 内部処理の最適化を行い、特に OpenCL 環境におけるレンダリングパフォーマンスの向上が顕著です。

v2.0b ​

  • [品質改善] メモリ使用量の最適化と安定性向上 — 搭載メモリの少ない環境で、再生や描画中にホストアプリケーションごと強制終了してしまうことがある問題に対処しました。内部のメモリ管理を見直し、長時間の使用や繰り返しの描画でもメモリ使用量が安定するよう改善しています。
  • [変更] プリセットの制限値の調整 — 各プリセットの最大文字数・最大パーツ数のバランスを見直しました。Low は複雑な文字も扱いやすいよう最大16文字 / 64パーツに、High はメモリ負荷を抑えつつ実用的な範囲となるよう最大60文字 / 180パーツに変更しています（Mid は従来どおり最大30文字 / 90パーツ）。引き続き preferences.json で各上限値をカスタマイズできます。

NOTE

プリセットの制限値はプラグインを新規に配置した際の初期値です。preferences.json をすでにカスタマイズしている場合は、その設定値が優先されます。

v2.0a ​

  • [新機能] 複数行テキストのサポート — テキスト内で改行（Enterキー）が入力可能になり、複数行のレイアウトに対応しました。また、Editページのインスペクタなど改行が直接入力しにくい環境向けに、文字列 \n や <br> による改行入力にも対応。これらを改行させずにそのまま文字として表示するためのエスケープ表示（\\n, \<br>）もサポートしました。
  • [新機能] 行間調整パラメータ（Line Spacing）の追加 — 複数行テキストの行間（上下の行の隙間）を調整できるようになりました。
  • [新機能] 水平・垂直アライメントの分離 — これまで1つだった揃え位置（Alignment）のパラメータを、水平方向（Horizontal Alignment）と垂直方向（Vertical Alignment）の2つに分割し、縦横それぞれの位置を個別に調整できるようにしました。
  • [新機能] 動作制限が異なる3つのプリセットプラグイン（Low / Mid / High）の提供 — 入力するテキストの長さや文字の複雑さに応じて選択できるよう、Low（動作最軽量）、Mid（標準）、High（長文・複雑文字向け）の3つのエフェクトが選べるようになりました。また、環境設定ファイル preferences.json を用いて、各上限値を任意にカスタマイズ可能です。
  • [新機能] 文字ごとの個別シャドウカラー — Character Settings ページで、各文字のシャドウカラーを個別に指定できるようになりました。Use Indiv Shadow Color を ON にすることで、グローバルのシャドウカラーを上書きして、文字ごとに異なるシャドウ色を設定できます。
  • [品質改善] プレビュー画面での回転ハンドルの操作性改善 — ビューア上のハンドルを使って文字やパーツを回転させる際、角度が急に不連続に飛んで（ジャンプして）しまう現象を修正しました。
  • [品質改善] 静的な手描き風エフェクト使用時の再生負荷軽減 — 手描き風エフェクトのパターン数（Pattern Count）を 1 に設定し、歪んだ状態のまま静止させたい場合に、プレビュー再生がカクつく現象を解消しました（ホスト側のフレームキャッシュが効くようになります）。
  • [品質改善] 全体的な安定性の向上 — 内部処理を見直し、さまざまな操作・設定の組み合わせにおける安定性を改善しました。

WARNING

破壊的変更: アライメントパラメータが Horizontal Alignment と Vertical Alignment に分離されたため、旧バージョンで作成されたプロジェクトをロードすると、揃え位置（Alignment）の設定がリセットされます。お手数ですが、再度アライメントを設定し直してください。

v1.6g ​

  • [改善] フォント列挙の改善 — バリアブルフォント（複数のウェイト・スタイルを持つフォント）を個別に検出し、リストに列挙・適用する機能を追加。あわせて各OSでのフォント走査処理を最適化し、フォントリスト構築の安定性と速度を向上。
  • [新機能] 輪郭線（ストローク）の接続スタイルとマイター制限の追加 — Global Settings ページに Join Style と Miter Limit パラメーターを追加。角の形状（Miter Clip / Round / Bevel / Miter Round / Miter Bevel）を柔軟に制御可能に。
  • [バグ修正] 塗りつぶしと輪郭線（ストローク）の描画順序の修正 — 塗りつぶしレイヤーとストロークレイヤーを分離して描画するよう修正。太いストロークを設定した際に、隣の文字やパーツが不自然に隠れて（削れて）しまう不具合を解消。

v1.6f ​

  • [新機能] 手描き風エフェクト（Hand Drawn Effect） — Animation ページに追加。輪郭線の波打ち（ウォブル）と文字の位置・回転・スケールのランダムなずれ（ジッター）を組み合わせた手書き風アニメーションを表現できます。
  • [バグ修正] グラデーション Okhsv/Okhsl の表示崩れ修正 — 白・黒・グレーなどの無彩色を含むグラデーションで Okhsv / Okhsl を使用した際に、色が正しく表示されない問題を修正。

v1.6d ​

  • [新機能] グラデーション（Gradient） — テキスト全体にグラデーションを適用する機能を追加。Font Color をスタート色として、Gradient End Color までを指定の Color Space で補間します。Color Space は Okhsv / Okhsl / Oklab / Linear RGB から選択でき、Angle・Start・End・Midpoint で方向と範囲を制御できます。
  • [品質改善] パフォーマンス向上・動作安定化 — 全体的な処理速度の向上と動作の安定化を図りました。

v1.6c ​

  • [品質改善] グリフのパーツ分解精度の向上 — 8 や複雑な漢字など一部のグリフでパーツ分解が正しく行われない問題を修正しました。
  • [品質改善] シャドウ有効時のパフォーマンス向上 — ドロップシャドウを有効にした際のレンダリング速度が改善されました。
  • [品質改善] フォント描画品質・互換性の大幅向上 — フォントの描画品質と互換性が飛躍的に向上しました。
  • [新機能] 合字（リガチャー）のサポート — fi や fl などの文字が美しく結合されて表示されるようになりました。
  • [品質改善] 縦書きの品質向上 — 日本語縦書き時の文字間隔・中央揃え、括弧 「」 や長音符 ー の表示が改善されました。
  • [品質改善] フォント名の日本語表示 — フォントリストで日本語名（例：ＭＳ ゴシック）が表示されるようになりました。

WARNING

破壊的変更: フォント名の取得方法が変わったため、既存プロジェクトを開いた際にフォント選択がずれる場合があります。その場合は Inspector でフォントを再選択してください。

v1.5d ​

  • [新機能] Alt+K ショートカットによるキーフレーム設定 — プレビュー画面上で Alt+K（macOS: Option+K）を押すことで、現在選択中の文字・パーツに対して現在フレームの値をキーフレームとして記録できる機能を追加。
  • [バグ修正] アニメーション設定時の個別設定（カラー・ストローク）の有効化不具合の修正 — キャラクターおよびパーツの個別カラー有効化フラグ（Use Indiv Color）と、キャラクターの個別ストローク有効化フラグ（Use Indiv Stroke）が、誤ってアニメーションのキーフレーム登録対象に含まれてしまっていたため、アニメーション設定時にこれらのフラグが正常に機能しない場合がある問題を修正。
  • [品質改善] UIラベルの更新 — Animation ページの「Set Keyframes for Selected」ボタンの表示名にショートカット表記 (Alt+K) を追加。

v1.5c ​

  • [品質改善] パフォーマンス向上 — 時間経過に伴うパラメータ変更処理中に、高負荷なUI更新や不要な再描画が発生して動作が重くなる問題を修正。

v1.5b ​

  • [新機能] キーフレーム最適化 (Optimize Keyframes) — Animation ページに「Optimize All Parameters」ボタンを追加。全パラメータから重複する冗長なキーフレームを自動検出して一括削除し、プロジェクトを軽量化する機能を追加。
  • [品質改善] システムフォント列挙のネイティブ化 — フォントリストの生成方式を、従来のファイルシステムスキャンから各OSのネイティブAPI（Windows: DirectWrite, macOS: CoreText, Linux: Fontconfig）による取得に刷新。網羅性と正確性が大幅に向上。
  • [品質改善] フォント名による同期 — 内部的にフォント名（fontName）での状態同期および名前解決を導入。新しいフォントの追加などでリスト順（インデックス）が変わっても、意図しないフォントに化けてしまう問題を解消。旧プロジェクトからのレガシーフォント自動移行機能も実装。

v1.5a ​

  • [新機能] Tracking — 全文字に均等に適用するグローバル文字間隔パラメーターを追加。
  • [新機能] Shadow Settings — ドロップシャドウ（Distance / Angle / Color）を追加。
  • [新機能] Separation 2 — Output Settings に第2分離グループ（Separation 2 Targets）を追加。Output Mode も3択に拡張。
  • [品質改善] 複雑な文字の描画精度向上 — 複数のパーツや穴（中空部分）を持つ複雑な文字で、塗りや重なりが正しく描画されない問題を改善。

WARNING

破壊的変更: 以下の変更は既存プロジェクトに影響する場合があります。

  • Alt+↑/↓ キー（Global モード） — 全文字の Margin After を増減する動作から、Tracking を増減する動作に変更。
  • Stroke > Order のデフォルト値 — Stroke Over Fill から Fill Over Stroke に変更。デフォルトのまま使用していた場合、レンダリング結果が変わります。
  • TTC フォントの全フェイス列挙 — .ttc ファイルに含まれる全フェイスが個別にフォントリストへ列挙されるようになりました。.ttc より後に列挙されるフォントを使用していた場合、フォント選択がずれる可能性があります。その場合は Inspector で再選択してください。
