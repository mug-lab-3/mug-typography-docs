How to Use ​

In the conventional 2D, 3D, and Lite presets, enable scripting as follows:

  1. Set Page to Scripting.
  2. Turn Script Enabled on.
  3. Enter a Lua script into Script Text, or use Paste Script to paste it from the clipboard.

In the Mug Typography Scripting preset, the Scripting group appears directly on the Global page, so no page switch is required. This preset omits Character Settings and Part Settings, supports up to 128 characters and 512 parts, and controls individual characters and parts from Lua. Use a conventional preset when you want to combine individual Inspector editing with Lua.

For example, this script alone keeps every character swaying with a per-character phase offset:

lua
function OnLayout(ctx)
    for index = 1, ctx.char_count do
        local character = ctx.chars[index]
        character.rotation = math.sin(ctx.time * 2.0 + index * 0.4) * 15.0
        character.offset_y = math.sin(ctx.time * 3.0 + index * 0.7) * 0.02
    end
end

The script overrides the inspector values every frame, so it combines freely with keyframes and other features such as deformers, Write On, and 3D.

The Three Callbacks ​

A script can define up to three functions. All of them are optional.

CallbackTimingTypical use
OnInitialize(ctx)Once when the script loadsPrecompute constants (store them in mt.storage)
OnPreLayout(ctx)Before layout, every frameChange base values: text, tracking, alignment, global colors, camera
OnLayout(ctx)After layout, every frameAdjust position, rotation, scale, and colors per character and part

The core model is two-staged: first the base values are settled, then the characters and parts generated from that base are adjusted. Changes that affect the character count (such as a typewriter effect) belong in OnPreLayout; moving individual characters belongs in OnLayout.

The mt.* namespace provides numeric utilities such as mt.clamp / mt.lerp / mt.remap, easing functions in mt.ease.*, color helpers in mt.color.*, and mt.storage for values saved at initialization.

The VSCode Simulator (MT Sim) ​

The distribution ZIP includes the VSCode extension Mug Typography Scripting Simulator (MT Sim) in the scripting-simulator/ folder. It lets you develop and preview scripts entirely without launching the host application.

  • Install: In the VSCode Extensions view, choose "Install from VSIX..." and select the bundled .vsix file.
  • Launch: Open a .lua file and run Mug Typography Scripting Simulator from the preview icon in the editor title or from the Command Palette. Saving the file reloads it instantly.
  • Copy to the plugin: Click Copy Script in the top toolbar, then click Paste Script in the plugin to replace Script Text.
  • Editing support: Completion and hover documentation for ctx.* and mt.*, filtered so each callback only offers the fields it can actually write.
  • Scene reproduction: Timeline playback, font selection, per-character and per-part inspectors, an evaluated-values view, and a camera preview for the 3D presets.

The simulator is an approximate preview

MT Sim approximates the plugin's layout and rendering. Gradient interpolation, projected stroke quality, and similar details are simplified, so always verify the final result in Mug Typography itself.

Samples and API Reference ​

The distribution ZIP ships many ready-to-use Lua samples in scripting-simulator/samples/:

  • reference/ — learning samples exercising every field of each API object and library
  • showcase/ — finished animation patterns: typewriter, bounce-in, neon glow, 3D camera moves, and more
  • diagnostics/ — scripts for verifying API behavior

For the complete API specification, see scripting-simulator/docs/scripting_api.md, also included in the distribution ZIP (Japanese).
