OFFSET ANIMATION

Turn one motion into a sequence

Duplicate a keyframe animation created on one character across other characters and parts with an automatic time offset. Just set a Frame Offset to quickly build cascading, follower-style motion where characters start moving one after another.

  • Propagate from one motion
  • Stagger with Frame Offset
  • Free ordering of targets
How to UseSettings Reference

How to Use ​

  1. Create a keyframe animation on one source character or part.
  2. Set Page to Copy / Paste.
  3. Select the character or part you animated in step 1 and click Copy Selection.
  4. Under Paste Target Attributes, enable the animation items to duplicate.
  5. Select multiple destination characters or parts in the viewer.
  6. Enter the number of frames between destinations in Frame Offset.
  7. Click Paste Animation with Offset.

With the copy source as the reference (no shift), the complete animation is duplicated onto the destinations with cumulative delays of 1×, 2×, ... the Frame Offset. A Frame Offset of 3 delays each successive destination by three frames; -3 staggers them in the opposite time direction.

IMPORTANT

Any animation already set on the destination characters and parts is erased for the checked parameters and replaced with the copied one.

Notes for the Fusion Page ​

The Fusion page in DaVinci Resolve does not support direct animation copying by OFX plugins, so the operation may take longer than on the Edit page and extra keyframes may be created.

IMPORTANT

On the Fusion page, after step 7 wait a few seconds for the keyframes to settle, then click Repair Animation Paste without changing the selection or checked targets. This removes keyframes placed at unexpected times and cleans up the offsets.

Specifying the Order ​

By default, offsets accumulate in logical index order (c1, c2, ...). List characters and parts in Order to accumulate offsets in that sequence instead.

text
c3 c1 c2

In the example above, character 3 starts first (Frame Offset × 1), character 1 next (Frame Offset × 2), and so on. The syntax matches Write On Manual Order, and ranges such as p2-p5 can also be used. Unlisted destinations follow in logical index order.

For the full specification (axis handling of 2D parameters, keyframe requirements for color channels, and more), see Copy / Paste.
