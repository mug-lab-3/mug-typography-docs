Release Notes

Track the updates and new features added to Mug Typography.

v5.0a 2026/7/20 ​

  • ✨ New Lua Scripting
    Added a Scripting page to the inspector. A Lua 5.4 script written in Script Text can control the text layout and the per-character / per-part position, rotation, scale, colors, and more on every frame. With the three callbacks OnInitialize / OnPreLayout / OnLayout plus utilities such as easing (mt.ease.*) and color helpers (mt.color.*), procedural animations that are hard to keyframe — waves, random scatter, and the like — take only a few lines of code (see Animate with Lua Scripts for details).
  • ✨ New Mug Typography Scripting Preset
    Added a dedicated preset that omits Character Settings and Part Settings and can control up to 128 characters and 512 parts from Lua. It supports the 3D camera, and its Scripting settings appear on the Global page. Lua is also available from the Scripting page in the conventional 2D, 3D, and Lite presets.
  • ✨ New Bundled VSCode Simulator "MT Sim"
    The distribution ZIP now includes the VSCode extension "Mug Typography Scripting Simulator" in the scripting-simulator/ folder. It previews and edits scripts without launching DaVinci Resolve, with completion and hover documentation for ctx.* / mt.*, timeline playback, and a camera preview for the 3D presets. Ready-to-use sample scripts and the API reference (docs/scripting_api.md) are bundled alongside it.

v4.1a 2026/7/17 ​

  • ✨ New Paste Animation with Offset
    Added Paste Animation with Offset, which duplicates the complete source animations onto the selected characters and parts while shifting each target by a cumulative Frame Offset. It also supports a custom expansion sequence through Order and Repair Animation Paste for removing unexpected keys added on the Fusion page.
  • ✨ New Add or Remove Multiple Selections
    In Character or Part mode, hold WinCtrl+Alt/MacCommand+Option and click to add a character or part to the current selection, or remove an already selected target.
  • ✨ New Camera Mode Shortcut
    With a 3D preset, press WinAlt+Ctrl+M/MacOption+Command+M to switch to Camera selection mode.
  • ✨ New Aggressive Keyframe Cleanup
    Added Aggressive Cleanup to Clean Up Redundant Keyframes, allowing middle keyframes to be removed from runs of unchanged values.
  • Quality Crayon Fill Quality
    Improved the Crayon scribble fill with pigment density variation and wax ridges for a more paper-like texture. The effect of End on outline overshoot and endpoint variation has also been refined.
  • Quality Font Identity Parameter Shown
    The Font Identity parameter, used internally to identify the current font, is now shown. Deinstance it together with Font to use a different font on each node split out with Output Separation (see Output Settings for details).

v4.0a 2026/7/12 ​

  • ✨ New 3D Typography
    Added the Mug Typography 3D and Mug Typography Lite (3D) presets, with Global and Character Yaw/Pitch, Character and Part Z Offset, and perspective projection through a dedicated camera.
  • ✨ New Camera Mode and Camera Controls
    Camera Yaw/Pitch, Target, Perspective Strength, Camera Zoom, and Lens Shift can now be adjusted directly in the viewer. Shift-drag axis locking is supported.
  • ✨ New Copy/Paste for 3D Parameters
    Selection copy/paste now includes Z Offset and the character-only Yaw and Pitch parameters.
  • Quality 3D Projection and Rendering
    The rendering pipeline now projects gradients, shadows, bounding boxes, and morph output in 3D, with improved gradient sampling quality.

v3.3b 2026/7/6 ​

  • ✨ New Copy / Paste Entire Instance Settings
    All parameters of an effect instance can now be transferred to another instance via the clipboard. (See Copy / Paste Entire Instance Settings for details.)

v3.3a 2026/7/5 ​

  • ✨ New Crayon — Hand-Scribbled Fill
    Added a Crayon mode to the gradient Color Space that draws hand-scribbled strokes over the normal fill. (See Crayon (Hand-Scribbled Fill) for details.)
  • ✨ New Bounding Box Fill Offset / Border Offset
    Added offset parameters to shift the background fill and border rectangles independently.
  • ✨ New Manual Order — Custom Reveal Order
    Write On can now reveal and draw characters/parts in any order you list, e.g. c3, c1, c2. (See Write On (Reveal Filter) for details.)
  • ✨ New Spaces are now selectable
    Glyph-less characters can now be selected and manipulated in Character mode via a virtual rectangle.
  • Changed Overlay Settings cleanup
    Removed the Unselected Color parameter and adjusted the default outline appearance to be subtler.
  • Bug Fix Vertical position of spaces in vertical text
    Improved the vertical origin calculation for glyph-less characters in vertical layout.
  • Quality Rendering performance optimizations
    Internal restructuring and improved caching for better preview responsiveness and memory efficiency.

v3.2j 2026/6/27 ​

  • Bug Fix Rare crash on reset
    Fixed an issue where DaVinci Resolve could occasionally crash when resetting parameters.

v3.2h / v3.2i 2026/6/27 ​

  • ✨ New Write On — Reveal Characters and Parts in Order
    Added the Write On feature to Output Settings. Just keyframe the Start / End values to make the characters or parts of your text appear or disappear in order, from beginning to end. Combine it with Stroke Order to easily create a "write-on" animation that draws the text stroke by stroke. (See Write On (Reveal Filter) for details.)

v3.2g 2026/6/22 ​

  • Bug Fix Deform / Hand Drawn targets now follow glyphs during a morph
    Fixed a bug where, during Morph Animation playback, the target specification (Targets, e.g. c2, p3) for the Path Deformation effects (Pulse / Trapezoid / Sphere / Barrel) and the Hand Drawn effect would be applied to unrelated parts instead of the intended characters/parts. Targets are now resolved against both the source (Text) and destination (To Text) layouts, so specifications like c2 / p3 correctly follow their glyphs throughout the morph.

v3.2f 2026/6/21 ​

  • Quality Improved Morph Matching Accuracy
    Improved part matching in Morph Animation so that characters with custom styling (such as colors) are preferentially matched to the character at the same position in the target text. This prevents styled characters from drifting to unintended parts during the morph, resulting in more natural transformations.

v3.2 (v3.2a / v3.2b / v3.2c / v3.2d / v3.2e) 2026/6/21 ​

  • ✨ New Morph Animation
    Added the "Morph Animation" feature, which smoothly morphs text A into text B. By keyframing the Progress parameter on the timeline, you can easily control the morphing speed and timing.
  • ✨ New "Baseline" Vertical Alignment
    Added a "Baseline" option to Vertical Alignment. This prevents the text from shifting up and down vertically when you edit text content, and ensures stable morph animations without vertical wobbling.
  • ✨ New Parameter Copy / Paste
    Added the ability to copy and paste transform parameters (position, rotation, scale, etc.) from one character/part to another. This greatly improves workflow efficiency when adjusting multiple elements.
  • ✨ New Outline Toggle Shortcut
    You can now quickly show or hide character outline borders in the preview screen using the WinAlt+;/MacOption+; shortcut.
  • Changed Independent Shadow Color Support
    Improved behavior when customizing shadow colors for specific characters (Use Indiv Shadow Color). Those characters will now cast shadows even if the global shadow setting (Draw Global Shadow) is turned OFF, making it much easier to highlight specific text.
  • Changed Expanded Value Range for Sphere / Barrel Effects
    The Amplitude range for "Sphere" and "Barrel" deformation effects has been expanded to -10.0 to 10.0 for numeric inputs, allowing for more dramatic distorting animations.
  • Quality Performance Optimizations
    Optimized rendering cache mechanisms and internal memory allocations to reduce preview stuttering, improving overall performance and memory usage during playback.

v3.1a 2026/6/14 ​

  • ✨ New Hollow Text (Transparent Fill)
    Added a Transparent Fill parameter to Global Settings. When turned ON, the text body (fill) becomes fully transparent and only the outline (Stroke) is rendered. Combined with a Stroke Width, you can create outline-only hollow text.
  • ✨ New Stroke Gap
    Added a Gap parameter that inserts a gap between the fill and the outline (stroke), to both the global outline settings and the per-character outline settings. This enables a "detached outline" look with three layers (body, gap, outline).
  • ✨ New Make Characters Transparent / Restore via Keyboard
    In Character selection mode, WinAlt+Ctrl+[/MacOption+Command+[ makes the selected characters transparent (clears alpha), and WinAlt+Ctrl+]/MacOption+Command+] restores transparency back to the global color. The key codes for these shortcuts can be customized via clearAlpha / restoreAlpha in preferences.json.
  • Changed Preset Max Part Adjustments
    Rebalanced the maximum part count for each preset to better manage memory load (Low: 64→60, Mid: 90→84, High: 180→170). The maximum character counts are unchanged (Low: 16, Mid: 30, High: 60). You can still customize each limit via preferences.json.

NOTE

The preset limits are the initial values applied when the plugin is newly placed. If you have already customized preferences.json, those settings take precedence.

v3.0c 2026/6/14 ​

  • Quality Smoother Path Deformation for Sphere & Barrel Effects
    Improved the path subdivision logic to use a fixed pixel width instead of a font-relative size. This ensures outline curves remain perfectly smooth without becoming jagged, even at very large font sizes or zoom levels.
  • Quality Enhanced Rendering Stability & Performance
    Optimized memory management for GPU (CUDA) rendering to improve overall stability and rendering performance.

v3.0a 2026/6/13 ​

  • ✨ New 4 New Path Deformation Effects
    Added animation effects that directly stretch and warp character outlines.
  • Pulse: Breathing-like expansion and contraction of the character outline outward and inward.
  • Trapezoid: Distortion that tapers the text into a trapezoid along a customizable angle axis.
  • Sphere: Inflates or pinches the characters into a spherical shape.
  • Barrel: Barrel or pincushion distortion that gradually changes the scale from the character's center toward the outside.
  • All deformers support speed, phase, per-character wave offsets, delay, and specific target parts (Targets).
  • ✨ New 3 Lite Presets (LiteLow, LiteMid, LiteHigh)
    Added lightweight presets that omit individual part parameters (Part Settings) to dramatically speed up plugin loading and UI rendering.
  • Benefits: Maximum character limits are automatically relaxed to 1.5x of the normal presets (LiteLow: 24, LiteMid: 45, LiteHigh: 90), and maximum parts are allowed up to the maximum drawable limit of 512 parts without any artificial limit.
  • ✨ New Align Selected Feature
    Added buttons on the Character Settings page to pack and align multiple selected characters, along with conflict-free default shortcuts WinAlt+Ctrl+H/MacOption+Command+H (horizontal align) and WinAlt+Ctrl+V/MacOption+Command+V (vertical align).
  • ✨ New Selection Mode Shortcuts
    Added default shortcuts WinAlt+Ctrl+A/MacOption+Command+A (Global), WinAlt+Ctrl+C/MacOption+Command+C (Character), and WinAlt+Ctrl+P/MacOption+Command+P (Part) to quickly toggle viewer selection modes. (Part mode is disabled in Lite presets).
  • [Improved] Customizable Keyboard Shortcuts
    You can now customize the basic keycodes for alignment, selection modes, and keyframe setting shortcuts by adding a shortcuts object in the preferences.json configuration file.
  • [Improved] Targets Parameter in Hand Drawn Effect
    Added the Targets parameter to the Hand Drawn effect table, allowing wobble/jitter to be applied only to specified parts or characters.
  • [Improved] Significant Performance Optimization
    Optimized internal processing logic, resulting in a dramatic performance boost, especially in OpenCL rendering environments.

v2.0b 2026/6/8 ​

  • Quality Memory Usage Optimization and Stability
    Addressed an issue where, on systems with limited memory, the host application could crash entirely during playback or rendering. Internal memory management was revised so that memory usage stays stable even during prolonged use or repeated rendering.
  • Changed Preset Limit Adjustments
    Rebalanced the maximum character and part counts for each preset. Low now allows up to 16 characters / 64 parts so that complex glyphs are easier to handle, and High was changed to up to 60 characters / 180 parts to keep memory load practical (Mid remains up to 30 characters / 90 parts). You can still customize each limit via preferences.json.

NOTE

The preset limits are the initial values applied when a plugin is newly placed. If you have already customized preferences.json, those values take precedence.

v2.0a 2026/6/8 ​

  • ✨ New Multi-line Text Support
    You can now write multi-line text by typing newlines. For environments where direct newlines are difficult to enter, such as the Edit page Inspector, typing \n or <br> tags is also supported. Added support for escape sequences (\\n and \<br>) to render these characters literally without line breaks.
  • ✨ New Line Spacing
    Added Line Spacing parameter to adjust the vertical gaps between lines in multi-line layouts.
  • ✨ New Horizontal & Vertical Alignment
    Split the single Alignment parameter into Horizontal Alignment and Vertical Alignment for independent axis control.
  • ✨ New Three Tiered Performance Presets (Low / Mid / High)
    Added three preset plugins with different text/part limits so you can choose the optimal version (Low for best performance, Mid for default typical usage, High for longer/complex texts). Limits can be customized globally via the preferences.json configuration file.
  • ✨ New Per-Character Shadow Colors
    Added the ability to set individual shadow colors for each character in the Character Settings page. Enable Use Indiv Shadow Color to override the global shadow color and apply unique shadow colors to specific characters.
  • Quality Smooth Rotation Interactions
    Fixed the issue where character or part rotation angles would abruptly jump when using preview viewer handles.
  • Quality Hand Drawn Performance for Static Wobble
    Fixed preview stuttering (stuttering playback) when Hand Drawn Pattern Count is set to 1 (static wobble). Host frame caching is now properly enabled in this state to avoid unnecessary redraw overhead.
  • Quality Overall stability improvements
    Revised internal processing to improve stability across a wide range of operations and setting combinations.

WARNING

Breaking Changes: Due to the split of the alignment parameter into Horizontal Alignment and Vertical Alignment, alignment settings in older projects will be reset. Please re-adjust alignment in the Inspector if necessary.

v1.6g 2026/6/5 ​

  • [Improved] Font Enumeration Improvements
    Added support for Variable Fonts to detect and list individual styles as independent fonts. Also optimized system font scanning on Windows, macOS, and Linux for faster and more stable list loading.
  • ✨ New Stroke Join Style and Miter Limit
    Added Join Style and Miter Limit parameters to Stroke Settings to control corner shapes (Miter Clip / Round / Bevel / Miter Round / Miter Bevel).
  • [Bug Fix] Stroke Draw Order Bug Fix
    Separated fill and stroke rendering passes to fix the issue where thick outlines would overlap and clip adjacent characters or parts.

v1.6f 2026/6/5 ​

  • ✨ New Hand Drawn Effect
    Added to the Animation page. Combines path wobble and per-character position/rotation/scale jitter to produce an organic, hand-drawn animation feel.
  • [Bug Fix] Gradient display glitch with Okhsv/Okhsl
    Fixed an issue where colors were not displayed correctly when using Okhsv or Okhsl with gradients containing achromatic colors such as white, black, or gray.

v1.6d 2026/6/4 ​

  • ✨ New Gradient
    Added gradient support for the entire text. Font Color serves as the start color, with a configurable Gradient End Color. The Color Space parameter selects the interpolation mode (Okhsv / Okhsl / Oklab / Linear RGB). Direction and range are controlled via Angle, Start, End, and Midpoint.
  • Quality Performance and stability improvements
    Overall rendering performance and operational stability have been improved.

v1.6c 2026/6/2 ​

  • Quality Improved glyph part decomposition accuracy
    Fixed an issue where some glyphs such as 8 and complex CJK characters were not decomposed into parts correctly.
  • Quality Better performance when Drop Shadow is enabled
    Rendering speed when using drop shadow has been improved.
  • Quality Font rendering quality and compatibility greatly improved
    Font rendering quality and compatibility have been dramatically improved.
  • ✨ New Ligature support
    Characters like fi and fl are now rendered as beautifully combined glyphs.
  • Quality Vertical writing mode improvements
    Character spacing, centering, and display of brackets 「」 and prolonged sound marks ー in vertical mode have been improved.
  • Quality Localized font names in the Inspector
    The font list now shows localized names (e.g., ＭＳ ゴシック).

WARNING

Breaking change: The font name resolution method has changed, so font selection may shift when opening existing projects. If this occurs, please reselect the font in the Inspector.

v1.5d 2026/5/31 ​

  • ✨ New Alt+K Shortcut for Setting Keyframes
    Pressing WinAlt+K/MacOption+K on the viewer screen sets the current parameter values as keyframes for the currently selected characters/parts (equivalent to "Set Keyframes for Selected") without modifying their values.
  • [Bug Fix] Issue with Individual Color/Stroke Activation when Keyframing
    Fixed an issue where the activation flags for individual colors (Use Indiv Color for characters and parts) and individual strokes (Use Indiv Stroke for characters) were incorrectly registered as keyframe parameters, which prevented these toggle states from being properly preserved and updated during animation.
  • Quality UI Label Update
    Added the shortcut reference (Alt+K) to the "Set Keyframes for Selected" button display name in the Animation page.

v1.5c 2026/5/28 ​

  • Quality Performance enhancement
    Fixed an issue where expensive UI updates and unnecessary redraws could occur during parameter changes over time, causing lagging operations.

v1.5b ​

  • ✨ New Keyframe Optimization (Optimize Keyframes)
    Added the "Optimize All Parameters" button to the Animation page, enabling bulk detection and removal of redundant keyframes to clean up project data.
  • Quality Native System Font Enumeration
    Replaced the file-system directory scanning with native system APIs (DirectWrite on Windows, CoreText on macOS, Fontconfig on Linux) to retrieve the font list. This significantly improves accuracy and coverage.
  • Quality Font Syncing by Name
    Introduced the fontName parameter for internal state syncing and name-based font resolution. This prevents font selection from shifting to an unintended font even when the list order changes. Includes automatic migration for legacy font configurations.

v1.5a ​

  • ✨ New Tracking
    Added a global character spacing parameter applied uniformly to all characters.
  • ✨ New Shadow Settings
    Added drop shadow support (Distance / Angle / Color).
  • ✨ New Separation 2
    Added a second separation group (Separation 2 Targets) to Output Settings. Output Mode now has three options.
  • Quality Rendering accuracy for complex characters
    Improved an issue where the fill and overlaps were not drawn correctly for complex characters with multiple parts or holes (hollow regions).

WARNING

Breaking Changes: The following changes may affect existing projects.

  • WinAlt+↑/↓/MacOption+↑/↓ (Global mode)
    Changed behavior from adjusting Margin After for all characters to adjusting Tracking.
  • Stroke > Order default
    Default changed from Stroke Over Fill to Fill Over Stroke. Projects using the default value will render differently.
  • TTC font face enumeration
    All faces in .ttc files are now listed individually in the font list. If you were using a font listed after a .ttc file, the font selection may have shifted. Please re-select the font in the Inspector if needed.
