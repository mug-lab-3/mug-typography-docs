Reset ​

ParameterDescription
Reset All CharactersResets Pos, Pivot, Scale, Stretch, Rotation, Margin After, Use Indiv Color, Color, Use Indiv Stroke, Stroke Width, Stroke Gap, and Stroke Color for all characters and parts to their default values.

NOTE

Multi-line Text Input and Line Breaks

  • Inputting on the Fusion Page: On the Fusion page, you can type newlines directly by pressing the Enter key to write multi-line text.

  • Inputting on the Edit Page: Due to Edit page Inspector UI limitations, you cannot type Enter to create newlines directly in the text box. If you want to write multi-line text on the Edit page, insert \n or <br> at the desired line break positions. These are automatically treated as line breaks.

    Also note that on the Edit page, pressing Enter while typing may move focus to a different parameter instead of inserting a line break. Be careful not to press Enter unintentionally while typing.

  • Escaping Line Breaks: If you want to render the literal characters \n or <br> without breaking the line, prepend them with a backslash to escape them: \\n or \<br>.

Morph Animation ​

A group for morph animation, located just below the Text field in Text Settings. It smoothly morphs the outlines of the string entered in Text into the outlines of another string — for example, making "A" melt into "B."

Basic Usage:

  1. Turn Enable Morph ON.
  2. Enter the starting string in Text.
  3. Enter the ending string in To Text.
  4. Keyframe Progress to go from 0 (starting text) to 1 (ending text).
ParameterDescriptionDefault
Enable MorphEnables morph animation. When ON, the normal text rendering is replaced by a morph from Text (start) to To Text (end).OFF
To TextEnter the destination text the morph ends at. Multi-line is supported.(empty)
ProgressSpecifies the morph progress from 0.0 (= the Text state) to 1.0 (= the To Text state). Keyframe this value on the timeline to freely control the speed and timing of the morph. Easing is also adjusted via this keyframe curve.0.0
Matching StrengthSpecifies how strongly the morph refines its part pairings beyond simple spatial proximity. At 0.0, parts are matched based purely on position (original behaviour). Higher values do two things: (1) they consider filled-area similarity, preferring to pair parts with similar painted area (e.g., avoiding a small dot pairing with a full stroke), and (2) they keep a character that has an individual color, stroke, or shadow paired with the character at the same position in the To Text, so its individual styling is not stranded onto an unrelated glyph.0.3
Same Glyph PrioritySpecifies the strength of pairing identical characters between the start and end texts. At 0.0, characters match purely by position. At 1.0, identical characters are always paired regardless of distance. Identical character pairs slide into place rather than changing shape, keeping the animation clean.0.3

NOTE

While Morph is ON, the normal text rendering is replaced by the dedicated morph rendering. The source text for the morph is taken from the Text field above.

WARNING

When the number of characters or parts differs between the start and end of the morph, per-character and per-part color, stroke, and shadow settings may not match how they appear with Morph turned OFF. For this reason, using individual per-character or per-part color, stroke, and shadow settings is not recommended with Morph animation.

TIP

Path Deformation effects (Pulse / Trapezoid / Sphere / Barrel) and the Hand Drawn effect can also be layered on top of the morph result.

TIP

You can morph even when the starting and ending texts have a different number of characters (excess characters automatically appear/disappear). However, texts with a similar number of characters and similar shape complexity (part count) morph more naturally and cleanly.

TIP

If part matching does not behave as intended, try changing Horizontal Alignment from Center to Left or Right. This changes the relative character positions and can change the matching result. For vertical movement, try Baseline under Vertical Alignment.

Gradient ​

A subgroup within the Color Settings group. Applies a gradient to the entire text using Font Color as the start color. The gradient is enabled when Color Space is set to any value other than None (the only exception is Crayon, which draws scribble strokes instead of a gradient; see below).

NOTE

The gradient range is calculated from the bounding box (BB) encompassing all parts at their transformed positions. If any character or part is moved or scaled such that the BB changes, the gradient range shifts across all characters accordingly.

ParameterDescriptionDefault
Color SpaceSelects the color interpolation mode for the gradient. None disables the gradient. Okhsv / Okhsl provide perceptually uniform, hue-preserving interpolation. Oklab interpolates in perceptual lightness space. Linear RGB uses linear RGB interpolation. Crayon is a special mode that, instead of a gradient, draws hand-scribbled strokes in the end color (Gradient End Color) over the normal fill (see below).None
Gradient End ColorThe end color of the gradient (RGB + Opacity). Font Color serves as the start color.Orange (#ff8000, opaque)
MidpointThe position of the 50/50 blend point between start and end colors (0.5 = equal split).0.5
AngleThe gradient direction in degrees. 0° is left to right, 90° is top to bottom.270°
StartThe gradient start position relative to the bounding box (0 = BB start, 1 = BB end).0.2
EndThe gradient end position relative to the bounding box (0 = BB start, 1 = BB end).0.8

Differences in Interpolation by Color Space ​

Even with the same two colors (start color #0000FF = blue, end color #FFFF00 = yellow), the appearance of the intermediate colors differs greatly depending on the chosen Color Space. Okhsv / Okhsl travel along the hue wheel through vivid intermediate colors (cyan to green), Oklab keeps perceptually uniform lightness, and Linear RGB produces a muddy gray midpoint due to simple numeric interpolation.

Crayon (Hand-Scribbled Fill) ​

When Color Space is set to Crayon, instead of applying a gradient, hand-drawn scribble strokes in the Gradient End Color are drawn over the normal fill (Font Color). This creates a texture that looks like the text was roughly colored in with a crayon or colored pencil.

In this mode, the gradient parameters are reinterpreted as follows:

ParameterMeaning in Crayon mode
Gradient End ColorColor of the scribble strokes
AngleStroke direction
MidpointSpacing between strokes
StartStroke thickness
EndStroke wobble (waviness) strength

Outline (Stroke) ​

Renders the outline (stroke) of the text glyphs. Useful when you want to keep text legible over busy backgrounds, or to give characters a bold, outlined look.

TIP

To make the outline visible, first set Width to a value greater than 0.0. The outline is not rendered while it remains at 0.0.

ParameterDescriptionDefault
WidthSpecifies the thickness of the outline as a ratio to the font size (e.g., 0.05 is about 5% of the character height). It does not scale with transform or stretch scaling. Setting it to 0.0 disables outline rendering.0.0
GapSpecifies the width of the gap between the fill and the outline (stroke) as a ratio to the font size. When set greater than 0.0, the area between the text body and the outline is cut out, producing a three-layer "detached outline" look (body, gap, outline).0.0
ColorSpecifies the color of the outline (RGB + Opacity).Off-black (#1c1c1e)
OrderSpecifies the draw order of the stroke and fill. Fill Over Stroke draws the fill on top of the stroke, so only the outer half of the stroke is visible, creating an outside-stroke effect. Stroke Over Fill draws the stroke on top of the fill, causing it to overlap the glyph and appear to bite inward.Fill Over Stroke
Join StyleSpecifies the join style for the corners of the stroke.
・Miter Clip: Miter join. Sharpens corners, but clips (flattens) them if the miter length exceeds the miter limit.
・Round: Rounds corners.
・Bevel: Flattens corners (beveling).
・Miter Round: Miter join, but rounds corners if the limit is exceeded.
・Miter Bevel: Miter join, but bevels corners if the limit is exceeded.		Miter Clip
Miter LimitThe limit value for corner sharpness in miter joins. Sharper angles result in longer miter lengths. If the length exceeds this limit, the fallback treatment specified in Join Style (clip, round, bevel, etc.) is applied.4.0

Bounding Box Settings ​

Configures rendering and filling of the bounding box (background frame) surrounding the entire text. Useful when you want to add a backing bar behind subtitles, or frame the text to give it a lower-third / telop look.

NOTE

The bounding box (BB) is the smallest rectangle that encloses all characters and parts. When you move characters or parts individually, the size and position of the BB updates accordingly.
