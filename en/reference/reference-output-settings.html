PARAMETER REFERENCE

Output Settings

Separate characters or parts into outputs, then control their visible range and reveal order for compositing and Write On.

⑂SeparationSplit output into up to three streams↘✎Write OnReveal characters or parts in sequence↘↕Manual OrderSpecify the reveal order explicitly↘{}TargetsSyntax for selecting characters and parts↘

Configures settings to isolate/extract only specific parts or characters as a separate output, or to render only select elements.
By using this feature, you can output only specific parts with a transparent background for multiple instances of the same text, enabling advanced compositing by applying different colors and effects to each part in subsequent nodes.

Parameters ​

ParameterDescriptionDefault
Enable Output SeparationEnables the output separation feature. When ON, the following parameters appear.OFF
Output ModeSpecifies how the separation targets are rendered. Select from the following three:
・Default: Renders only the parts not included in either Separation 1 or Separation 2 targets (bounding box background is also rendered).
・Separation 1 Only: Renders only the parts specified in Separation 1 Targets (bounding box background is not rendered).
・Separation 2 Only: Renders only the parts specified in Separation 2 Targets (bounding box background is not rendered).		Default
Separation 1 TargetsA text input field for specifying the characters/parts to include in the first separation group. For the notation, see "Targets Format" below.
If left empty, this separation group contains nothing (i.e. the group has no targets).		(empty)
Separation 2 TargetsA text input field for specifying the characters/parts to include in the second separation group. Uses the same format as Separation 1 Targets (an empty field contains nothing).(empty)

IMPORTANT

When duplicating an instanced node on the Fusion page for output separation, deinstance Output Mode after duplication. Then set one node to Default and the other to Separation 1 Only or Separation 2 Only. If it remains instanced, Output Mode changes on both nodes together.

IMPORTANT

For separation targets (Separation 1 / 2 Targets), an empty field means the group contains nothing (no targets). Note that this is the opposite of the Targets field for the Path Deformation / Hand Drawn effects described later, where an empty field means everything is targeted.

TIP

Workflow Example

  1. Copy the effect node to create three instances.
  2. On the first node, turn Enable Output Separation ON, set Output Mode to Default, specify c1 in Separation 1 Targets and c2 in Separation 2 Targets (renders everything except the 1st and 2nd characters).
  3. On the second node, turn Enable Output Separation ON, set Output Mode to Separation 1 Only, and specify c1 in Separation 1 Targets (renders only the 1st character).
  4. On the third node, turn Enable Output Separation ON, set Output Mode to Separation 2 Only, and specify c2 in Separation 2 Targets (renders only the 2nd character).
  5. Apply different effects to each node and merge them together to create animations where each character has its own distinct effect.

TIP

Using a different font per output
To use a different font on each node you split out with Output Separation, deinstance both Font and Font Identity (found just below it) in Text Settings together. These two parameters work together to preserve the current font, so deinstancing only one of them will not restore the correct font. Once deinstanced, you can pick an independent Font on each node.

Write On (Reveal Filter) ​

Write On is a filter that progressively reveals or hides the characters or parts of your text, from beginning to end.
By keyframing the Start and End parameters on the timeline, you can easily create a "handwriting" animation where characters appear one by one.

ParameterDescriptionDefault
UnitSelects the unit for the filter.
· Character: Filters by character. Only characters within the Start–End range are rendered.
· Part: Filters by part. Combine with Stroke Order to reveal parts in an order close to the writing order.		Character
StartSets the start of the visible range as a fraction from 0.0 (beginning) to 1.0 (end). Animatable.0.0
EndSets the end of the visible range as a fraction from 0.0 (beginning) to 1.0 (end). Animatable.1.0
Stroke OrderWhen ON, reorders each character’s parts so they reveal in a natural-looking order. Especially effective when Unit is Part — parts appear in roughly stroke-writing order, producing a write-on effect.OFF
Manual OrderWhen ON, the characters/parts listed in the Manual Order List are revealed and drawn first, in the order listed. Anything not listed follows afterwards in the normal order.OFF
Manual Order ListA text field (multi-line) listing characters/parts in the order they should reveal and draw. Uses the same syntax as the "Targets Format": c<n> (character), p<n> (part), and ranges such as c1-c3 (note: since this expresses an order, the exclusion prefix ! is not supported). If the same part appears more than once, its first occurrence wins.(empty)

TIP

When to use Manual Order Write On normally reveals text in a fixed order, from the first character to the last. With Manual Order you can make characters appear in any order you like, e.g. c3, c1, c2. When combined with Stroke Order, the parts expanded from a c<n> token follow the stroke-order sequence.

NOTE

When both Start and End are at their default values (Start = 0.0, End = 1.0), the Write On filter is inactive (everything is shown).

IMPORTANT

About the Stroke Order reveal sequence
The reordering done by Stroke Order does not reproduce the actual, correct writing order of a character. It uses an internal algorithm to estimate a plausible order from the positions of the parts, so it may differ from the real stroke order. Also, how a character splits into parts depends on the font's data structure (see Overview), so the reveal order can change when you switch fonts, even for the same character. Preview with your actual font to confirm the order is what you intended.

Targets Format ​

The Separation 1 / 2 Targets of Output Separation and the Targets field of the Path Deformation and Hand Drawn effects all use the same notation to specify the target characters/parts.

List one or more tokens (units of specification) separated by commas (,) or spaces. Each token can use the following notations (case-insensitive — c1 and C1 are the same):

NotationMeaningExample
c<n>Character unit. Specifies all parts that compose the n-th character (1-based).c1 = all parts of the 1st character
p<n>Part unit. Specifies the n-th part globally (1-based).p5 = the 5th part
<n>No prefix. Same as p<n>; specifies the n-th part.15 = the 15th part (same as p15)
<a>-<b>Range. Specifies everything from a to b, inclusive. See below for details.c1-c3, p2-p5, 2-5
!<token>Negation (exclusion). Prefixing a token with ! excludes the targets it would otherwise select. See below.!c2, !p3, !2-5

Details of range specification (-) ​

  • Both endpoints of a range must be the same kind (character-to-character or part-to-part). Mixed-kind ranges (e.g. c1-p3) are invalid and ignored.
  • You only need to add a prefix to one endpoint. The endpoint without a prefix inherits the kind of the other endpoint.
    • c1-3 is the same as c1-c3 (characters 1–3). You don't have to write out c1-c3 in full.
    • 1-3 is the same as p1-p3 (parts 1–3).
    • Inheritance works in reversed order too: c3-1 becomes c3-c1 → sorted to c1-c3.
  • Even if the start and end are reversed (e.g. c3-c1), the range is automatically sorted into ascending order.

Details of negation (!) ​

  • A token prefixed with ! removes its targets from the result. It works with ranges and character units too (e.g. !c2, !p3-p5).
  • Token order does not matter. All inclusions and exclusions are accumulated, and then exclusion wins (if the same part is both included and excluded, it is dropped from the targets).
  • Negation is handy in combination with each effect's "empty behavior" (see below).

Empty-field behavior (important — opposite per feature) ​

What an empty field means differs (is the opposite) depending on the feature.

FeatureMeaning when empty
Path Deformation / Hand Drawn effects (Targets)All characters/parts are targeted.
Output Separation (Separation 1 / 2 Targets)Nothing is included (no targets).

Because of this, for the Path Deformation / Hand Drawn effects, if you want to target "everything except specific parts," it is convenient to write only a negation. For example, entering !c1 targets everything except the 1st character (starting from "empty = all," with just the 1st character excluded).

Examples ​

InputInterpretation
c1, p12, 15All parts of the 1st character, the 12th part, and the 15th part
c1-c3All parts of characters 1–3
c1-3Same as c1-c3 (characters 1–3). The prefix on the end can be omitted
p2-p5 / 2-5Parts 2–5 (both are the same)
c1-5 !c3All of characters 1–5 except character 3
!p3 (on a deformation effect)All parts except the 3rd part

NOTE

Invalid specifications are silently ignored (no error is raised): values <= 0, indices out of range, mixed-kind ranges, and so on.
