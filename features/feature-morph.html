MORPH & TRANSITION

流れるように言葉が言葉に変わる

元のテキストから別のテキストへ、文字のかたちを自動で対応づけて滑らかに変形。Progressへキーフレームを2つ置くだけで、文字が溶け合うトランジションを作れます。

  • キーフレーム2つで完成
  • 自然な変形アルゴリズム
  • 字数違いに対応
使い方を見る設定リファレンス

使い方 ​

  1. Page を Global Settings にします。
  2. Text に変形元の文字列を入力します。
  3. Morph Animation 内の Enable Morph をオンにします。
  4. To Text に変形先の文字列を入力します。
  5. 開始フレームで Progress を 0 にしてキーフレームを設定します。
  6. 終了フレームで Progress を 1 にします。

0 は Text、1 は To Text の形です。中間の値では両者のパーツを補間します。キーフレームの間隔とカーブで速度感を調整してください。

連続して変形させる ​

A→B→C のように、3つ以上の文字列を続けて変形させることもできます。

  1. 最初のモーフが完了したフレームで、Text を To Text と同じ文字列に変更してキーフレームを設定し、Progress を 0 に戻します。
  2. 同じフレームで To Text も次の文字列に変更してキーフレームを設定し、再び Progress を 1 へアニメーションさせます。

Progress が 1 のときの表示と、Text を差し替えて 0 に戻した直後の表示は同じ形になるため、切り替えは画面上では目立ちません。これを繰り返すことで、いくつでも変化を連鎖できます。

対応づけを調整する ​

文字数や形が大きく異なるときは、内部で「どのパーツからどのパーツへ変形するか」を推定します。

パラメーター調整する場面
Matching Strength面積の近さを重視して対応づけたいとき。0 では位置の近さを優先します。
Same Glyph Priority両方に同じ文字があるとき、その文字同士を優先して結びたいとき。

きれいに見せるコツ ​

  • まずデフォルト値で動きを確認し、パーツが不自然に交差するときだけ Matching Strength などを少しずつ変更します。
  • 同じ文字を同じ位置に残したい場合は Same Glyph Priority を上げます。
  • 形が大きく異なる文字列では、Progress の中間を短くすると変形の複雑さが目立ちにくくなります。
  • 縦方向の位置が揺れる場合は Vertical Alignment の Baseline を試します。
  • 対応づけが思い通りにならないときは、Horizontal Alignment を Center から Left や Right に変更してみます。文字の位置関係が変わるため、対応づけの結果も変化します。

NOTE

パーツの分かれ方はフォントのアウトライン構造に依存します。同じ文字列でも、フォントを変えるとモーフの対応や動きが変わることがあります。
